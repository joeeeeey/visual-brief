## Recommended: Story (Chinese)

```text
我最近一直在想：AI 协作里卡住的，很多时候不是资料不够，而是我脑子里的上下文，别人没法快速看懂，也没法放心验证。

所以做了 Visual Brief：把关键截图/图解、对应的 canonical link，和一小段面向读者的解释放进同一个包里。

它能写决策回复、带图操作指南和公开技术解释。

截图降低检查成本，链接保留追问和反驳的路径。
https://github.com/joeeeeey/visual-brief
```

## Q&A (Chinese)

```text
为什么很多“我已经调研过了”的结论，还是很难被采纳？

因为读者往往只能看到结论：不知道最关键的证据在哪，也不知道该去哪里核对。

我把这个工作流做成了 Visual Brief：
- 图紧贴它支持的主张
- 事实保留 canonical link
- 生成图只做说明，不冒充证据
- 支持决策回复、操作指南、公开解释

https://github.com/joeeeeey/visual-brief
```

## Story (English)

Screenshots reduce verification effort. Canonical links preserve accountability.
